# Changes - 2026-03-03

## Summary
This update delivered reliability, security, observability, and deployment hardening across Tekton + ArgoCD, and introduced a clearer GitOps structure and promotion flow.

## CI/CD Reliability
- Added Tekton pipeline resilience:
  - task `retries` for clone/test/integration/scan/build
  - PipelineRun `timeouts` for `pipeline`, `tasks`, and `finally`
  - `finally` summary/notification step with optional `notification-webhook-url`
- Added Argo deploy failure handling in `scripts/k8s/start-argo.sh`:
  - improved error extraction from Argo Application status
  - optional deploy notification webhook (`--notify-webhook-url`)
- Added post-deploy smoke testing + rollback controls in `start-argo.sh`:
  - smoke test path (`--smoke-path`, default `/healthz`)
  - optional skip (`--skip-smoke`)
  - auto-rollback on rollout/smoke failure (default on, disable via `--no-auto-rollback`)
- Added CI/CD summary utility:
  - `scripts/k8s/cicd-status.sh` (Tekton + Argo health summary; optional webhook)

## Promotion Flow (dev -> staging -> prod)
- Introduced GitOps base/overlay structure:
  - `manifests/gitops/base/`
  - `manifests/gitops/overlays/dev`
  - `manifests/gitops/overlays/staging`
  - `manifests/gitops/overlays/prod`
- Added promotion script:
  - `scripts/k8s/promote.sh`
  - updates `newTag` in target overlay
  - keeps `manifests/environments/<env>.env` `IMAGE_TAG` in sync when present
- Updated Argo deploy script to support `--env dev|staging|prod` and default app path to `manifests/gitops/overlays/dev`.

## Security
- Added External Secrets + Vault scaffolding:
  - `manifests/security/external-secrets/cluster-secret-store-vault.example.yaml`
  - `manifests/security/external-secrets/tekton-secrets.externalsecret.example.yaml`
- Added Cosign signing/verification scaffolding:
  - `scripts/k8s/sign-image.sh`
  - `manifests/security/cosign/cosign-public-key.secret.example.yaml`
  - `manifests/security/kyverno/verify-cosign-policy.example.yaml`

## Observability Quality
- Added structured JSON logging with trace IDs:
  - new `logger.js`
  - wired request logging and error logging into `app.js` and `server.js`
  - logs now include `trace_id` from active OTel context
- Added Grafana provisioning for dashboards + alerts:
  - dashboard provider config
  - CI/CD + app overview dashboard (logs, restarts, sync/pipeline metrics)
  - alert rules for app error logs, pod restarts, Argo sync failures, Tekton pipeline failures
- Expanded Prometheus/Thanos scraping:
  - kube-state-metrics deployment/service/RBAC
  - ArgoCD metrics target
  - Tekton metrics targets

## App/Runtime Hardening
- Added sample-node-app runtime hardening in split app manifest:
  - readiness/liveness probes (`/healthz`)
  - resource requests/limits
  - pod anti-affinity (preferred)
  - HPA (`autoscaling/v2`)
  - PodDisruptionBudget
- Added split app manifest path:
  - `manifests/apps/sample-node-app/`

## Local Registry Flow
- Added local image preload automation:
  - `scripts/k8s/registry-preload.sh`
  - loads image into Docker Desktop kind nodes for reliable pulls
- Added local registry retention cleanup:
  - `scripts/k8s/registry-retention.sh`
  - keeps newest N tags and deletes older manifests (when registry delete API is enabled)

## GitOps Hygiene
- Split manifests into:
  - `manifests/apps/`
  - `manifests/platform/`
  - `manifests/gitops/base`
  - `manifests/gitops/overlays/*`
- Updated documentation references for the new layout and deploy flow.

## Testing Gates

  - `npm run test:integration`
- Added Tekton integration-test gate before scan/build:
  - new task `manifests/tekton/tasks/npm-integration-test.yaml`

## Additional Governance/Security Follow-up (same day)
- Added GitOps path drift preflight in `scripts/k8s/start-argo.sh`:
  - verifies `--revision` contains `--path` before applying Argo `Application`
  - prevents `ComparisonError` from unresolved/missing path on tracked branch
- Added rollback mode controls in `start-argo.sh`:
  - `--rollback-mode kubernetes|gitops` (default `kubernetes`)
  - GitOps mode skips `rollout undo` and points to GitOps rollback script
- Added GitOps-native rollback script:
  - `scripts/k8s/gitops-rollback.sh` (revert commit touching overlay, optional push, hard refresh Argo app)
- Hardened promotion governance in `scripts/k8s/promote.sh`:
  - required explicit approval for staging/prod (`--approved`)
  - optional remote push verification (`--verify-push`)
  - Tekton success precheck + in-cluster smoke precheck before promotion (unless `--skip-checks`)
- Added enforceable security bootstrap:
  - `scripts/k8s/start-security.sh` installs ESO + Kyverno and applies security manifests
  - concrete (non-example) manifests added under `manifests/security/` for Vault store, ExternalSecrets, and Kyverno image verification policy (`Enforce` by default)
- Added Helm-based environment variant:
  - `charts/sample-node-app` with `values-dev.yaml`, `values-staging.yaml`, `values-prod.yaml`
  - `scripts/k8s/start-helm-app.sh` for Helm deployment path

## Finalization Updates (same day, later)
- Completed Vault auth bootstrap and runtime wiring:
  - Added auth-mode specific SecretStore manifests for `token`, `approle`, `jwt`
  - Added bootstrap scripts:
    - `scripts/k8s/vault-bootstrap-token.sh`
    - `scripts/k8s/vault-bootstrap-approle.sh`
    - `scripts/k8s/vault-bootstrap-jwt.sh`
  - Extended `scripts/k8s/start-security.sh` with `--vault-auth-mode` and corresponding auth args.
- Completed supply-chain enforcement path:
  - Added Tekton `cosign-sign` task (`manifests/tekton/tasks/cosign-sign.yaml`)
  - Wired pipeline/triggers/runs to execute image signing before ArgoCD sync
  - Added signed-image gate in `start-tekton.sh` (`COSIGN_SIGN_ENABLED` and secret precheck)
  - Added Kyverno PolicyException strategy example:
    - `manifests/security/kyverno/policy-exception-sample-node-app.example.yaml`
- Added policy/testing gates in CI:
  - New `policy-gates` GitHub Actions job (`.github/workflows/ci.yml`)
  - Kustomize render + kubeconform schema checks
  - Conftest policy checks (`policy/conftest/main.rego`)
  - Kyverno CLI tests (`policy/kyverno/**`)
- Clarified secret storage model:
  - Vault (`kv/ci/*`) is source-of-truth.
  - ESO syncs runtime Kubernetes Secrets (`docker-credentials`, `sonarqube-credentials`, `cosign-key`) into namespace `default`.
  - Auth bootstrap secrets are in `external-secrets` namespace (`vault-token` / `vault-approle`).
  - Optional Kyverno key helper secret in `kyverno/cosign-public-key`.
- E2E fix for local registry rollout:
  - Added/validated preload path using `scripts/k8s/registry-preload.sh` when new tag pull fails on nodes.
